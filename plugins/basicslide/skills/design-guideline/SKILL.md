# Design Guideline for Slides

The goal is presentations that are **memorable and intentional**, not merely correct. A bold direction executed with precision always beats a safe direction executed perfectly. Refined minimalism and dramatic maximalism are both valid — the key is **intentionality**.

This skill has three reference files:

- **[judgment.md](judgment.md)** — Bold philosophy, perceptual + strategic principles, anti-patterns. Read when designing or evaluating.
- **[rules.md](rules.md)** — Color, typography, spacing, layout, deck composition, editorial rules, accessibility baseline. Read when looking up tokens or thresholds.
- **[patterns.md](patterns.md)** — HTML layout patterns and Marp CSS rendering constraints. Read when picking a structural layout or hitting a CSS rendering limitation.

Token values source of truth: `themes/basicslide.css`.
